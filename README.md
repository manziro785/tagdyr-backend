# Tagdyr Backend (SimuLife 2.0)

Серверная часть игры-симулятора жизни. Главный принцип: **симуляцию считает клиент,
сервер хранит и проверяет**. Движок детерминирован (seeded RNG), поэтому шарится
между клиентом и сервером — это даёт анти-чит реплеем.

## Структура

```
tagdyr-backend/
├── packages/
│   ├── engine/    # чистый TS-движок: reduce, seeded RNG, скип времени, индекс жизни
│   ├── schemas/   # zod-схемы домена и DTO API — единый источник типов
│   └── content/   # данные событий/персонажей/концовок (валидируются schemas)
└── apps/
    └── api/       # Hono + Drizzle + PostgreSQL
```

> Фронтенд живёт отдельным репозиторием. `packages/engine` и `packages/schemas`
> публикуются как npm-пакеты и переиспользуются фронтом — отсюда изоморфная
> валидация и серверный реплей.

## Стек

- **Рантайм:** Node.js (LTS)
- **API:** Hono · JWT (bearer)
- **БД:** PostgreSQL (Render) · Drizzle ORM · postgres-js
- **Валидация:** zod (shared)
- **AI:** Vercel AI SDK (фаза 6)
- **Тесты:** Vitest

## Команды

```bash
pnpm install
pnpm test          # юнит-тесты всех пакетов
pnpm typecheck     # проверка типов
pnpm dev           # запуск API локально
pnpm db:generate   # сгенерировать миграцию по схеме Drizzle
pnpm db:migrate    # применить миграции
```

## Фазы

Реализация по фазам из ТЗ (бэкенд стартует с фазы 2 «Сезонность»). Каждая фаза —
задеплоенный инкремент; не начинаем следующую, пока текущая не в проде.
